# Peter Griffin TTS API Contract

## Endpoint

```
POST /generate_speech
```

Deployed via Modal as a FastAPI web endpoint. The URL is assigned by Modal on deploy (e.g. `https://<workspace>--peter-griffin-chatterbox-generate-speech.modal.run`).

## Authentication

Include `api_key` in the JSON body. The value must match the `TTS_API_KEY` environment variable set via the `tts-api-key` Modal Secret.

Unauthorized requests receive a `401` response.

## Request

```json
{
  "api_key": "string (required)",
  "text": "string (required) - the script text to synthesize",
  "job_id": "string (required) - unique job identifier, used as R2 path prefix",
  "reel_index": "integer (required) - reel number within the job, used in R2 filename"
}
```

### Example

```json
{
  "api_key": "your-secret-key",
  "text": "So thermodynamics is basically the study of how energy moves around, and let me tell you, it is way more interesting than it sounds.",
  "job_id": "abc123",
  "reel_index": 0
}
```

## Response

### 200 OK

```json
{
  "success": true,
  "audioKey": "audio/abc123/0.wav",
  "timestampsKey": "timestamps/abc123/0.json",
  "durationSeconds": 6.42,
  "wordCount": 25
}
```

### 400 Bad Request

```json
{
  "error": "text is required"
}
```

Returned when `text`, `job_id`, or `reel_index` is missing or invalid.

### 401 Unauthorized

```json
{
  "error": "unauthorized"
}
```

### 502 Bad Gateway

Returned if the TTS generation fails internally.

## R2 Output Files

Both files are written to the `learnscrolling-assets` R2 bucket, mounted at `/root/r2` inside the Modal container.

### Audio file

- **Key**: `audio/{job_id}/{reel_index}.wav`
- **Format**: WAV (PCM)
- **Details**: Generated by Chatterbox TTS (Peter Griffin voice, exaggeration=0.7), sped up 1.5x via sox tempo effect (no pitch shift)

### Timestamps file

- **Key**: `timestamps/{job_id}/{reel_index}.json`
- **Format**: JSON

```json
{
  "durationSeconds": 6.42,
  "wordTimestamps": [
    { "word": "So", "start": 0.0, "end": 0.18 },
    { "word": "thermodynamics", "start": 0.18, "end": 0.84 },
    { "word": "is", "start": 0.84, "end": 0.96 }
  ]
}
```

Each entry in `wordTimestamps`:
- `word` (string): the spoken word, stripped of whitespace
- `start` (float): start time in seconds, relative to the audio file
- `end` (float): end time in seconds, relative to the audio file

Timestamps are generated by running Whisper `base` on the sped-up audio, so they correspond exactly to the delivered WAV file.

## Modal Secrets Required

| Secret Name | Keys | Purpose |
|---|---|---|
| `huggingface-secret` | `HF_TOKEN` | Download Chatterbox model weights |
| `tts-api-key` | `TTS_API_KEY` | Authenticate incoming API requests |
| `r2-secret` | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY` | Mount the Cloudflare R2 bucket via S3-compatible API |

## Environment Variables

| Variable | Where to set | Purpose |
|---|---|---|
| `CF_ACCOUNT_ID` | System env or hardcode in `app.py` | Cloudflare account ID for the R2 endpoint URL (`https://<id>.r2.cloudflarestorage.com`). Not a secret — it's in every R2 public URL. |

## Pipeline

1. Chatterbox Turbo TTS generates raw audio from text (Peter Griffin voice clone)
2. Sox applies 1.5x tempo speedup without pitch shift
3. Whisper `base` extracts word-level timestamps from the sped-up audio
4. Both WAV and timestamps JSON are written directly to R2 via cloud bucket mount
5. R2 keys are returned in the response for the Cloudflare Workflow to reference downstream
